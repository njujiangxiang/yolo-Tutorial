# 数据准备指南

> 从零开始准备 YOLO 训练数据的完整流程

---

## 📋 数据准备流程总览

```
确定检测目标 → 收集数据 → 数据清洗 → 数据标注 → 数据增强 → 质量检查
     ↓             ↓           ↓           ↓           ↓           ↓
  明确需求     50-100 张/类   去模糊去重   LabelImg    2-5 倍增强   最终验证
```

---

## 步骤 1：明确检测目标

### 定义检测类别

在开始之前，明确你要检测什么：

**示例：PCB 缺陷检测**
| 类别 ID | 类别名称 | 说明 | 示例特征 |
|--------|----------|------|----------|
| 0 | hole | 孔洞 | 圆形缺失 |
| 1 | scratch | 划痕 | 线状痕迹 |
| 2 | short | 短路 | 异常连接 |
| 3 | open | 断路 | 线路中断 |
| 4 | spurious | 多余物 | 异物附着 |

### 定义数据要求

| 项目 | 要求 | 理由 |
|------|------|------|
| 每类图片数 | ≥50 张 | 保证模型能学习特征 |
| 图片尺寸 | ≥640x480 | 保证小目标可见 |
| 角度覆盖 | 至少 3 种角度 | 提高泛化能力 |
| 光照变化 | 明暗各占 20% | 适应不同环境 |

## 步骤 2：收集数据

### 方法对比

| 方法 | 耗时 | 质量 | 成本 | 推荐度 |
|------|------|------|------|--------|
| 公开数据集 | 短 | 高 | 免费 | ⭐⭐⭐⭐⭐ |
| 网络爬虫 | 中 | 中 | 免费 | ⭐⭐⭐⭐ |
| 自己拍摄 | 长 | 高 | 设备成本 | ⭐⭐⭐⭐ |
| 数据合成 | 中长 | 中 | 时间成本 | ⭐⭐⭐ |

### 推荐策略

**第 1 天：快速启动**
1. 先下载公开数据集（30 分钟，50 张）
2. 同时用爬虫收集补充（1 小时，30 张）
3. 整理目录结构（15 分钟）

**第 2-3 天：针对性补充**
1. 分析缺失的场景/角度
2. 针对性拍摄或搜索
3. 达到每类 50+ 张

## 步骤 3：数据清洗

### 必做检查

```bash
# 1. 删除模糊图片
python examples/03-remove-blur.py --threshold 100

# 2. 删除重复图片
python examples/04-remove-duplicates.py

# 3. 生成质量报告
python examples/05-data-quality-check.py
```

### 质量指标

| 指标 | 合格值 | 检查方法 |
|------|--------|----------|
| 模糊度 | ≥100 | Laplacian 方差 |
| 亮度 | 50-200 | 平均灰度值 |
| 尺寸 | ≥200x200 | 直接检查 |
| 重复度 | <5% | 感知哈希 |

## 步骤 4：数据标注

### LabelImg 安装

```bash
# 安装
pip install labelImg

# 启动
labelImg
```

### 标注步骤

1. **打开图片目录**
   - 点击 `Open Dir`
   - 选择 `datasets/cleaned/`

2. **设置标注格式**
   - 点击 `Change Save Dir`
   - 选择 `datasets/labels/`
   - 格式选择 `YOLO`

3. **创建类别文件**（可选）
   ```
   # classes.txt
   hole
   scratch
   short
   open
   spurious
   ```

4. **开始标注**
   - 按 `W` 画矩形框
   - 选择类别
   - 按 `Ctrl+S` 保存
   - 按 `D` 下一张

### 标注快捷键

| 快捷键 | 功能 |
|--------|------|
| W | 画框 |
| D | 下一张 |
| A | 上一张 |
| Ctrl+S | 保存 |
| Ctrl+R | 改变框颜色 |
| 双击框 | 编辑框 |
| Delete | 删除框 |

### 标注质量要求

- ✅ 框要紧贴目标边缘
- ✅ 不要框住背景
- ✅ 小目标也要标注（≥10x10 像素）
- ✅ 遮挡目标标注可见部分
- ❌ 不要框太大（包含过多背景）
- ❌ 不要框太小（漏掉目标边缘）

### 标注检查

```bash
# 检查标注质量
python examples/check-annotations.py

# 可视化标注
python examples/visualize-labels.py
```

## 步骤 5：数据增强

### 缺陷检测推荐配置

```python
# 基础增强（必须）
A.HorizontalFlip(p=0.5)      # 水平翻转
A.VerticalFlip(p=0.5)        # 垂直翻转
A.RandomRotate90(p=0.5)      # 旋转 90 度

# 光照增强（可选，如果颜色不重要）
A.RandomBrightnessContrast(
    brightness_limit=0.15,
    contrast_limit=0.15,
    p=0.3
)

# 模糊增强（模拟不同焦距）
A.Blur(blur_limit=3, p=0.15)

# 锐化增强（增强缺陷边缘）
A.Sharpen(alpha=(0.2, 0.5), p=0.3)
```

### 增强倍数建议

| 数据量 | 增强倍数 | 说明 |
|--------|----------|------|
| <100 张 | 5 倍 | 小数据集需要更多增强 |
| 100-500 张 | 3 倍 | 中等数据量 |
| >500 张 | 1-2 倍 | 大数据集少量增强即可 |

### 运行增强

```bash
python examples/09-batch-augmentation.py
```

## 步骤 6：数据划分

### 推荐比例

```
训练集：70%  - 用于模型学习
验证集：20%  - 用于调参和早停
测试集：10%  - 用于最终评估
```

### 运行划分

```bash
python examples/06-split-dataset.py
```

### 分层抽样

确保每个类别在各集合中的比例相同：

| 类别 | 总数 | 训练集 | 验证集 | 测试集 |
|------|------|--------|--------|--------|
| hole | 50 | 35 | 10 | 5 |
| scratch | 60 | 42 | 12 | 6 |
| ... | ... | ... | ... | ... |

### 检查清单

- [ ] 目录结构正确
  ```
  dataset/
  ├── images/
  │   ├── train/
  │   ├── val/
  │   └── test/
  ├── labels/
  │   ├── train/
  │   ├── val/
  │   └── test/
  └── data.yaml
  ```

- [ ] 图片和标注数量匹配
- [ ] 标注格式正确（5 个值，归一化）
- [ ] data.yaml 存在且正确
- [ ] 类别分布平衡

### 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 标注值>1 | 未归一化 | 重新运行转换脚本 |
| 缺少标注 | 漏标 | 补充标注 |
| 类别不平衡 | 某类数据少 | 针对性收集 |
| training loss 不降 | 标注错误 | 检查标注质量 |
